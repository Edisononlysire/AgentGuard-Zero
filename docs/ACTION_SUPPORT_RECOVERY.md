# Action-Support Bootstrap Recovery Protocol

Status: **recovery implementation under review; no recovery training has been
launched**.

This document is the authoritative training policy after the July 2026 failure
audit. The former pure on-policy three-round run is retained only for failure
analysis. It is not a valid formal lineage.

## Why the lineage was reset

The audit identified two distinct failures:

1. Base and VDA1 preserved the JSON contract but had negligible support for
   probe, trust, memory, and mitigation actions.
2. Further PPO updates caused the VDA2/VDA3 output contract to collapse. DCA2
   and DCA3 were therefore trained from invalid feedback.

The recovery protocol establishes action support before RL, protects the action
contract throughout training, and permits a new DCA only after model-independent
and model-dependent gates pass.

`Zero` now means **zero human-labelled optimal defence actions**. Bootstrap
targets are generated by the simulator and a public-state robust teacher; no
human action label is used.

## Public-State Robust Teacher

For each public observation, the teacher enumerates finite public candidates:

- Observe;
- passive verification;
- active probes;
- trust operations;
- memory operations;
- mitigation responses.

Every argument must reference an identifier already present in the public
observation. The teacher clones a group of hidden worlds sharing the same public
state, admits an action only when all worlds admit it, and maximizes the worst
short-horizon utility across that group. Hidden state is used only to calculate
offline candidate utility. It is forbidden from prompts and targets.

Observe is always evaluated. A non-Observe candidate is selected only when its
robust advantage is strictly greater than `0.05`; equality and floating-point
tolerance resolve to Observe.

The SFT parquet contains public prompt/target records only. Candidate Q-values,
hidden-world hashes, and hidden-state checks are written to a separate audit
artifact.

## Executable gates

### Stage 0: environment and reward learnability

Run 200 canonical scenarios with Oracle, Public-State Teacher, Random Legal,
No-op, and Overreact. The gate requires:

```text
Oracle > Teacher > Random Legal > No-op > Overreact
Teacher Safe Utility - No-op Safe Utility >= 0.20
Teacher attack mitigation > 40%
```

Any failure stops the pipeline before model training. The implementation does
not relax thresholds to make an existing environment pass.

### Gate A: dual-arm Bootstrap SFT

Both arms use the same 400 scenarios, 2,000--3,000 public teacher records, and
hyperparameters:

```text
Qwen3.5 base + Bootstrap SFT
VDA1          + Bootstrap SFT

1 epoch; LoRA r=16, alpha=32; LR=1e-5; effective batch=64;
warmup=3%; weight decay=0.01
```

Each arm is evaluated with K=1 greedy on the same disjoint 200-scenario gate
(80 original + 120 new canonical). Required metrics include validity >=99%,
actionable Observe <=70%, active probes >=10%, mitigation >=15%, safe success
>=10%, positive probe yield, trust/memory operations >=5%, and invalid state
operations <=2%.

Mitigation, probe yield, or trust/memory coverage equal to zero is a hard no-go.
Actionable Observe above 80% is also a hard no-go. Only a passing arm may become
the new VDA_boot.

### One DAgger correction pass

After Gate A, collect the selected model's visited public states on 500 new
canonical scenarios, relabel them with the robust teacher, and add 1,000--3,000
correction records for one short SFT pass. Only one correction pass is allowed
before Gate B.

### Gate B: ten-step RL pilot

Gate B uses 160--320 scenarios, ten RL steps, adaptive G=2 to G=4 sampling,
20% bootstrap replay, and non-zero KL (`0.02`). Degenerate all-success,
all-failure, all-Observe, zero-variance, or malformed multi-turn groups are
resampled once and then excluded if still degenerate.

Gate B requires Safe Utility improvement over its SFT baseline, non-decreasing
mitigation, positive probe yield, validity >=98%, and actionable Observe <=80%.
It only unlocks teacher review; it does not automatically launch full training.

## Contract and collapse protection

- Valid canonical JSON receives exactly zero format reward. Invalid or
  incomplete JSON is rejected and penalized.
- Probe reward requires public information gain or a downstream grounded state
  or authorization change.
- Observe is penalized only when the robust teacher finds advantage >0.05.
- Training must retain bootstrap replay and non-zero KL.
- Rollout-only constrained decoding is forbidden unless the identical token
  mask is used for policy log-probabilities.
- A periodic dev check stops and rolls back on contract, action-support,
  mitigation, probe-yield, safe-success, or advantage collapse.

Full static-skill RL, a fresh DCA0, DCA-VDA co-evolution, and ECRG/V5-C
calibration remain locked until the preceding gates and a separate review pass.
V5-C is deployment governance and may not select a failing Train-only model.

## Review-only commands

Generate deterministic canonical worlds and run Stage 0:

```bash
python scripts/generate_recovery_canonical.py \
  --scenario-count 200 \
  --output /path/to/recovery_stage0.json

python scripts/run_recovery_stage0.py \
  --scenarios /path/to/recovery_stage0.json \
  --output /path/to/stage0_audit.json \
  --workers 4
```

Build public-only Gate-A records after Stage 0 passes:

```bash
python scripts/generate_recovery_canonical.py \
  --scenario-count 400 \
  --group-offset 1000 \
  --output /path/to/recovery_gate_a_train.json

python scripts/build_recovery_bootstrap.py \
  --scenarios /path/to/recovery_gate_a_train.json \
  --output-dir /path/to/recovery_bootstrap
```

Gate evaluators consume frozen metric JSON and never submit training:

```bash
python scripts/evaluate_recovery_gates.py gate-a \
  --base-metrics /path/to/base_gate_a.json \
  --vda1-metrics /path/to/vda1_gate_a.json

python scripts/evaluate_recovery_gates.py gate-b \
  --metrics /path/to/gate_b.json \
  --baseline /path/to/selected_gate_a.json
```

The recovery configuration is frozen in
`configs/recovery/action_support_bootstrap_v1.json`.
